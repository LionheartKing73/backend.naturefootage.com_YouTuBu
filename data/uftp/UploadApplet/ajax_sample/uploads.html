Persistent FileCatalyst Web Uploads

This sample page shows that you may navigate freely through an Ajax-powered website without interrupting a FileCatalyst transfer. All visible elements (including status area and its contents on bottom right) are created using pure HTML and CSS. To proceed, first configure “ajax_sample/fcupload.js” with the hostname (“server”), username (“user”), port, and password (“pass”). Once correctly configured, the intended scenario is as follows:

  1. Click Show Uploads on the transfer status area (bottom right).
  2. Click Browse and Add to Queue to select a file for transfer.
  3. Click Display Queue Items to confirm that the file is indeed queued for transfer.
  4. Click Upload. The FileCatalyst applet will connect to the FileCatalyst Server and the transfer will begin.
  5. While the transfer is still active, use the main navigation to go the other sample pages. New content is loaded via Ajax and the transfer is not interrupted.
  6. With the transfer active, you may also:
    • Cancel the transfer. Once canceled, click Upload again. The transfer will restart or resume depending on how much data has been sent (determined by a configurable block size)
    • Click Browse and Add to Queue again to add another file. "1 of 1" in the status area will increment to "1 of 2"; clicking Display Queue Items will show the new file added.
